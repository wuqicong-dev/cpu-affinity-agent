# 鲲鹏CPU架构与细粒度绑核优化Agent参考手册

# 一、鲲鹏CPU概述

鲲鹏CPU系列是华为自主研发的高性能ARM架构服务器CPU，采用**NUMA（非一致性内存访问）**架构，完美适配“CPU调度+NPU加速”的异构协同计算需求，能够为AI训练、推理等场景提供高效、稳定的算力支撑。

# 二、内存层次与拓扑架构

## 2.1 内存层次关系及访问场景

内存层次涵盖L1缓存、L2缓存、L3缓存、DDR内存及SCCL模块，各组件层级关联清晰，访问时延逐级递增，不同访问场景对应不同层级的资源调用，图中标注1-6的具体解释如下：

- **标注1**：访问L1缓存，最贴近CPU核心的缓存访问，时延最低

- **标注2**：访问L2缓存，层级介于L1与L3之间，承接L1缓存访问需求

- **标注3**：访问L3缓存，同域内CPU核心共享缓存，用于域内数据交互

- **标注4**：访问隔壁SCCL域下的L3缓存，跨域访问，时延高于同域L3缓存

- **标注5**：访问当前SCCL下的DDR内存，本地内存访问场景

- **标注6**：跨片访问场景，时延最低但波动最明显，属于特殊高速访问场景

## 2.2 Cluster拓扑结构

Cluster内部采用多核共享缓存的拓扑设计，核心缓存分配规则：

- 每个CPU Core内置独立的**L1I（一级指令缓存）**、**L1D（一级数据缓存）**，并绑定专属L2缓存

- 每4个CPU Core组合为一个单元，共享同一组L3缓存

## 2.3 核心缩写与概念解析

|缩写|全称|概念描述|
|---|---|---|
|CPU Core|处理核心|Kunpeng 920系列支持16/20/24/32/40/48/64核配置，每个核心独占L1、L2缓存|
|CPU Uncore|非处理核心|每个非处理核心独占L3缓存，负责缓存调度与数据同步|
|SCCL|super core cluster|包含6个CCL、2个ICL、4个DDRC及对应L3缓存，独立封装在一个CPU Die中，等同于CPU Die|
|CCL|core cluster|CPU核集群，由4个CPU Core和专用L2 Cache组成基础计算单元|
|SICL|Super IO Cluster|IO Die简称，包含多个ICL模块，负责IO资源统筹|
|ICL|IO Cluster|IO集群，物理位置相近、共享系统总线与内部接口的设备组合|
|DDRC|内存控制器|单个Die含4个DDRC，对应4条独立DDR通道，单通道支持2根DIMM内存|
|NUMA|非一致性内存访问|多CPU共享内存架构，每个CPU拥有本地内存，可跨CPU访问内存；具备独立DDRC的CPU Die为单独NUMA节点|
# 三、细粒度绑核优化核心前提

**绑核核心逻辑顺序**：厘清业务进程及进程名 → 深入分析进程内部线程行为 → 结合业务特征制定绑核策略

- 所有绑核操作围绕**识别模型推理、计算调度相关核心线程**开展

- 该方法论仅适用于**有明确标识的线程**；未命名线程无法直接套用细粒度绑核方案

# 四、PTA相关线程及细粒度绑核规范

## 4.1 PTA核心线程职责说明

|线程名称|核心职责|热点等级|
|---|---|---|
|mainThread|Pytorch+PTA主线程，负责PTA前向算子下发|第一热点|
|npuGuardThread|负责PTA反向算子下发|第二热点|
|aclThread|负责PTA二级流水（task queue）调度|第三热点|
|releaseThread|负责PTA资源释放|非热点|
|hcclCommWatchdogThread|负责PTA的HCCL通信监控|非热点|
|unknownThread|Pytorch线程池线程，数据并行处理自动调用|非热点|
## 4.2 细粒度绑核方式与优化结果

|线程类型|绑核方式|优化效果|
|---|---|---|
|热点线程（mainThread）|固定绑定绑核区间第1个CPU核，**禁止线程迁移**|减少核切换，提升缓存命中率，降低前向算子延迟|
|热点线程（aclThread）|固定绑定绑核区间第2个CPU核，**禁止线程迁移**|稳定流水任务，减少调度阻塞，提升任务执行效率|
|其他热点线程（npuGuardThread）|按顺序固定绑定绑核区间后续CPU核|避免热点线程核切换，保障反向算子计算性能|
|非热点线程（dataloader、unknownThread）|绑定绑核区间剩余CPU核，与热点线程**物理隔离**|降低CPU资源竞争，减少核切换损耗，保障热点线程优先级|
|特定场景业务线程|按需分析线程特征，灵活分配CPU核|适配专属业务场景，最大化系统整体算力利用率|
**Agent执行注意事项**

- 绑核前务必先甄别线程名称与热点等级，严禁混淆热点与非热点线程绑核区间

- 鲲鹏NUMA架构下，优先绑定同SCCL/NUMA节点内CPU核，避免跨片、跨域访问

- 未命名线程禁止强制绑核，需先做线程溯源，否则会引发系统调度异常

- 绑核后需监控缓存命中率、线程时延，验证优化效果
